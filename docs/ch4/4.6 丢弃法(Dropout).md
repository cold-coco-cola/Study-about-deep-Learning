# 4.6 丢弃法
丢弃法在训练中使用，增加波动（随机性），但保证期望保持不变，从而保证在推理时概率不会翻倍（假设p=0.5）

我尝试用一种更加直观的方法理解：现在你的面前有一个水果拼盘，包含的水果种类很多，有很多沙拉酱之类的东西。我们忽略你吃的过程，或者这样想：你昨天吃了水果拼盘，你记得吃的时候还不错（或者不太好），这有可能是因为哈密瓜、西瓜很甜，导致你只记得了哈密瓜、西瓜还不错，而忘记了其他水果和酱很一般，这就导致你对这个水果拼盘的评价很大程度上因为哈密瓜和西瓜造成了偏差。如果我们每次都吃同一个人做的水果拼盘，但是每次随机选择一些水果不放进去（dropout），多次之后就能限制哈密瓜、西瓜的影响，让你对自己对水果拼盘的喜爱程度有一个准确的判断。

反之，如果每次都把所有水果都摆进去，那这个其实也一般的水果拼盘会被你误认为挺好吃，你在推荐给朋友吃的时候，朋友反响平平。



操作方式：

其中h视作某个隐藏层输出，‘干掉’某几个神经元，让这些神经元别慌~冷静~

解释丢弃法：可以理解为正则项，限制某些参数。随机将某几个输出项置0。

丢弃概率**p**为超参数

下面是从零代码实现。考虑两个隐藏层的MLP

```python
import torch
from torch import nn
from d2l import torch as d2l
from py_proj import functions_needed as fcn

def dropout_layer(X, dropout):
    assert 0<= dropout <=1
    if dropout == 1 :
        return torch.zeros_like(X)

    if dropout == 0:
        return X

    mask = (torch.randn(X.shape) > dropout).float()
    #随机在(0,1)产生数，与dropout比较，得到bool值张量，.float转为1或者0
    return mask * X / (1.0 - dropout)
```

`mask * X`用乘法而非“选择”实现置0，这样能够提升运行速度，减少内存占用？

```python
num_inputs, num_outputs, num_hiddens1, num_hiddens2 = 784, 10, 256, 256

dropout1, dropout2 = 0.2, 0.5

class Net(nn.Module):
    def __init__(self, num_inputs, num_outputs, num_hiddens1, num_hiddens2,
                 is_training = True):
        super(Net, self).__init__()
        self.num_inputs = num_inputs
        self.training = is_training
        self.lin1 = nn.Linear(num_inputs, num_hiddens1)
        self.lin2 = nn.Linear(num_hiddens1, num_hiddens2)
        self.lin3 = nn.Linear(num_hiddens2, num_outputs)
        self.relu = nn.ReLU()

    def forward(self, X):
        H1 = self.relu(self.lin1(X.reshape((-1, self.num_inputs)))) #X是总的特征矩阵，重塑成二维张量，第二维是num_inputs.操作后得到每个样笨的特征矩阵？
        # 只有在训练模型时才使用dropout
        if self.training == True:
            # 在第一个全连接层之后添加一个dropout层
            H1 = dropout_layer(H1, dropout1)
        H2 = self.relu(self.lin2(H1))
        if self.training == True:
            # 在第二个全连接层之后添加一个dropout层
            H2 = dropout_layer(H2, dropout2)
        out = self.lin3(H2)
        return out


net = Net(num_inputs, num_outputs, num_hiddens1, num_hiddens2)
```

```python
num_epochs, lr, batch_size = 10, 0.5, 256
loss = nn.CrossEntropyLoss(reduction='none')
train_iter, test_iter = d2l.load_data_fashion_mnist(batch_size)
trainer = torch.optim.SGD(net.parameters(), lr=lr)
fcn.train_ch3(net, train_iter, test_iter, loss, num_epochs, trainer)
```

```python
net = nn.Sequential(nn.Flatten(), #记得flatten
        nn.Linear(784, 256),
        nn.ReLU(),
        # 在第一个全连接层之后添加一个dropout层
        nn.Dropout(dropout1),
        nn.Linear(256, 256),
        nn.ReLU(),
        # 在第二个全连接层之后添加一个dropout层
        nn.Dropout(dropout2),
        nn.Linear(256, 10))

def init_weights(m):  #生成权重
    if type(m) == nn.Linear:
        nn.init.normal_(m.weight, std=0.01)

net.apply(init_weights);
```
