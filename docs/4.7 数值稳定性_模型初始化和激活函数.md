# 4.7 数值稳定性，模型初始化和激活函数

更新参数的时候，用的是最终的损失l关于某个层参数W（暂且忽略b）的梯度，这个梯度是由多层的链式求导（反向传播）进行矩阵乘法而来。于是在逐层的求导中，梯度矩阵某些位置的乘数总是大于1，导致累乘到这一层时，产生了一个相当大的数，即**梯度爆炸**（反之则**梯度消失**）。

这显然是不合理的，并且产生了如下问题：
1. **梯度爆炸**
    1. 值问题：数太大（长）了，超出值域
    2. 学习率敏感：
    
        if lr 过大->大参数->大梯度
        
        else 训练慢
        
        无论如何，都会导致过多去调整学习率
2. **梯度消失**
    1. 值问题同上

    2. 训练慢
    
    3. 由于反向传播，底部（输入层方向）梯度小，训练效果差，只有顶部得到较好训练，从而使神经网络变“浅”。

>下面是F(X)对x求导得到对角矩阵的计算展示：

>  ∂F(x)/∂x =


> 【【∂f(x1)/∂x1, ∂f(x2)/∂x1, ∂f(x3)/∂x1】

> 【∂f(x1)/∂x2, ∂f(x2)/∂x2, ∂f(x3)/∂x2】

> 【∂f(x1)/∂x3, ∂f(x2)/∂x3, ∂f(x3)/∂x3】】

举个例子，回忆一下sigmoid函数：

```python
%matplotlib inline
import torch
from d2l import torch as d2l

X = torch.arange(-8.0, 8.0, 0.1, requires_grad=True)
y = torch.sigmoid(X)

y.backward(torch.ones_like(X))

d2l.plot(X.detach().numpy(), [X.grad.numpy(), y.detach().numpy()],
         legend=['sigmoid', 'gradient'], figsize=(4.5, 2.5))
```

靠近0时梯度太小，容易造成梯度消失。

**怎么解决上面出现的问题呢？**

我们想办法让让训练稳定;

首要的矛盾：梯度值

解决方案：让梯度值在合理范围
    
    1. 乘法变加法（resnet，lstm）
    
    2. 梯度归一化or梯度裁剪 
    
    3. 本次讨论的方法：合理权重初始化and激活函数

**让每层的方差是一个常数**：

就是让最后的loss对每一层的输出的梯度期望和方差相等，让这些梯度保证在一个范围内（我们希望他是合理的，不引起上述出现的梯度问题）


下面进行**权重初始化**：

在合理的范围内随机初始化权重:(需要概率论相关知识)

注意独立性的利用，我目前推了正向传播的初始化条件：
> n(输入层)γ(输出层)=1
反向推到会得到类似结果（矩阵求导还没有很清楚）
> n(输出层)γ(输出层)=1

显然，让两个层参数量相等是强人所难的。那么，我们尝试取γ为他们的平均，进行：
